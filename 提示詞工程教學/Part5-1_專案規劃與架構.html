專案介紹

星海粒子互動系統

一個結合「太空星塵」和「深海微光」主題的互動視覺專案。使用者透過手勢控制畫面上的數千顆粒子，創造出如星空或深海般的夢幻視覺效果。

專案特色

手勢互動

使用 MediaPipe Hands API 追蹤手部動作，無需額外硬體

粒子視覺

20,000+ 粒子即時渲染，流暢的視覺效果

主題切換

太空星塵 / 深海微光 兩種主題風格

純前端

HTML + CSS + JavaScript，無需後端伺服器

技術選型

MediaPipe Hands

用途：手部追蹤
特點：Google 開源、瀏覽器端運行、21 個關鍵點追蹤
優勢：免費、效能好、易於整合

Canvas 2D API

用途：粒子渲染
特點：原生 JavaScript、即時繪圖
優勢：學習曲線低、相容性好

requestAnimationFrame

用途：動畫循環
特點：瀏覽器優化的動畫 API
優勢：效能好、自動適應螢幕更新率

為什麼選這些技術？

這個技術組合讓專案可以在任何現代瀏覽器中運行，不需要安裝任何軟體，適合教學展示和學生練習。

用 AI 規劃專案架構

專案規劃提示詞

我想開發一個手勢控制的粒子互動網頁專案。

專案需求：
1. 使用 MediaPipe Hands 追蹤手部位置
2. 建立可容納 20,000 顆粒子的粒子系統
3. 手指位置會吸引/排斥周圍粒子
4. 支援「太空星塵」和「深海微光」兩種主題
5. 純前端，單一 HTML 檔案

請協助我：
1. 設計專案架構
   - 主要模組劃分
   - 各模組的職責
   - 模組間的資料流

2. 規劃開發步驟
   - 建議的開發順序
   - 每個階段的目標

3. 技術細節建議
   - MediaPipe 初始化配置
   - 粒子資料結構設計
   - 效能優化策略

用 AI 生成基礎程式碼

程式碼生成提示詞

請幫我建立星海粒子互動專案的基礎 HTML 結構。

要求：
1. 單一 HTML 檔案，包含 CSS 和 JavaScript
2. HTML 結構：
   - 隱藏的 video 元素（用於攝影機）
   - 全螢幕 canvas 元素
   - 控制面板（主題切換按鈕、粒子數量滑桿）

3. CSS 樣式：
   - 深色背景
   - canvas 全螢幕
   - 控制面板固定在右上角，半透明背景

4. JavaScript 基礎結構：
   - 全域變數宣告（particles, handPosition, currentTheme）
   - 主題配色定義（太空：藍紫色系，海洋：青藍色系）
   - 初始化函數框架
   - 動畫循環框架

請生成完整的基礎程式碼，加上詳細註解說明每個部分的用途。

AI 輔助開發技巧

有效的程式碼生成策略

將大專案拆成小模組，一次只請 AI 生成一個功能，確保每個部分都能正常運作後再整合。

分步驟請求範例

  • 第一步：「請生成粒子初始化的程式碼，產生 20000 顆粒子，每顆有 x, y, vx, vy, color, size 屬性」
  • 第二步：「請生成粒子物理更新的函數，包含速度衰減和邊界反彈」
  • 第三步：「請生成 Canvas 繪製粒子的函數」
  • 第四步：「請生成 MediaPipe Hands 的初始化和回呼處理程式碼」

Debug 協助

「這段程式碼出現錯誤 [錯誤訊息]，請幫我找出問題並修正」

效能優化

「這個粒子更新函數效能不佳，請幫我優化」

專案安裝與執行

取得專案程式碼

從 GitHub 複製專案到本地電腦

1

複製專案

將 <your-username> 替換為實際帳號：

git clone https://github.com/<your-username>/NeonFlux.git
cd NeonFlux
2

啟動本地伺服器

使用 Python 3 啟動：

# 使用 Python 3
python3 -m http.server 8000
3

開啟瀏覽器

訪問以下網址：

http://localhost:8000

注意事項

  • 需要 HTTPS 或 localhost 環境才能存取攝影機
  • 確保瀏覽器有攝影機存取權限
  • 建議使用 Chrome 或 Edge 瀏覽器
